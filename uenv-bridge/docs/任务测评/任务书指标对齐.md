# 任务书指标对齐

**一、数据集的选择**

1. 最好与ai4s有关

2. 我们的课题是主要做后训练—\>尽量少做/不做sft—\>如果选取的数据集太偏门冷门，模型在sft阶段没有见过这些数据，会导致rl效果不好—\>不能选择过于偏门冷门的ai4s数据集

3. 结题的指标是训练一套模型权重在10个任务上都超过任务专有模型—\>选择的数据集的sota指标不能过高，要有刷榜空间\&不能选任务专用模型刷榜太高的领域（比如遥感图像分类，遥感图像不是传统rgb图像，它由多种光谱图层合成，遥感领域的模型具有特殊架构，和vlm对比不公平）

4. 多数据集混训—\>数据集的类型选择和比例会影响效果—\>数据集选择要慎重\&尽量挑不需要环境/环境容易实现的，减少返工成本\&rollout轨迹都要收集好，方便以后调整比例再次训练

5. 相较于llm\-as\-a\-judge，尽可能选择基于规则评测的数据集，降低评测成本

**二、模型的选择**

1. 既要能做纯文本任务，又要能做多模态任务

2. 适合 16×A100 80GB 做后训练



**Qwen/Qwen3\.6\-35B\-A3B**

训练精度：BF16；显存不足时启用 QLoRA fallback

主训练方式：LoRA，rank=64 主线，rank=128 上限实验

阶段1（26\.11月的指标，无需视觉模态）：文本、代码、表格路径训练；视觉输入路径不更新

阶段2（27\.5月的指标，多模态）：打开视觉输入；训练 vision adapter \+ language LoRA



**三、训练方式的选择**

**Eval\-first \+ RL\-first**

> best\-of\-N ：同一道题生成多个候选答案，再选择验证器评分最高的结果；
> 
> 验证器过滤 ：用规则评测器、测试程序或执行环境自动筛掉错误输出；
> 
> rejection SFT ：只用通过验证的成功样本继续监督微调模型；
> 
> 格式 warm\-up SFT ：少量预热训练，主要让模型先学会合法输出格式，而不是大规模注入新能力。
> 
> 

流程：

```Plain Text
1. 先对 10 个任务类型分别运行 Qwen 原始基线。
2. 统计输出可解析率、验证器可打分率、正分样本率、执行成功率和 SOTA gap。
3. 达到 gate 的任务跳过 SFT，直接进入 best-of-N、验证器过滤、rejection SFT 和 GRPO/RLVR。
4. 未达到 gate 的任务只做极小规模格式 warm-up SFT，目标是让输出可解析、代码可执行、工具调用合法、patch 可应用。
5. 主要训练预算投入 verifier-driven rejection SFT 和 GRPO/RLVR。
```

任务级 gate：

```Plain Text
直接进入 RL / rejection SFT：
输出可解析率 ≥ 90%，验证器可打分率 ≥ 95%，best-of-N 正分样本率 ≥ 5%。

只做格式 warm-up SFT：
输出可解析率 < 90%，或工具/action/patch/表格 claim verification成功率 < 90%。

先做 teacher / 成功轨迹 warm-up：
best-of-N 正分样本率 < 1%，且执行环境确认正常。
```
